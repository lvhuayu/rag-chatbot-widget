🤖 RAG Chatbot Usage Examples

⚠️ Important: Make sure to configure the backendUrl to point to your actual RAG backend server!

🔐 Authentication Examples

1. Using JWT Token (Recommended)

Provide a JWT token directly for authentication:

initRAGChatbot({ backendUrl: 'https://your-backend.com', auth: { token: 'your-jwt-token-here', tokenKey: 'my_chatbot_token' } });

2. Public/Private Key Authentication (Advanced)

Use cryptographic keys for secure authentication without passwords:

initRAGChatbot({ backendUrl: 'https://your-backend.com', auth: { useKeyAuth: true, keyAuthEndpoint: '/auth/request-challenge', keyAuthVerifyEndpoint: '/auth/verify-challenge', keyStorageKey: 'my_chatbot_keys', username: 'your-username' // Optional } });

3. Auto-Login with Credentials

Automatically login with username and password:

initRAGChatbot({ backendUrl: 'https://your-backend.com', auth: { username: 'your-username', password: 'your-password', autoLogin: true, loginEndpoint: '/api/auth/login' } });

4. Using Stored Token

If you've previously authenticated, the token will be automatically retrieved:

initRAGChatbot({ backendUrl: 'https://your-backend.com', auth: { tokenKey: 'my_chatbot_token' // Will look for stored token } });

5. No Authentication (Limited Features)

Use without authentication (some features may be limited):

initRAGChatbot({ backendUrl: 'https://your-backend.com' // No auth config - will work but with limited access });

🎯 Complete Configuration Example

Full Featured Setup

initRAGChatbot({ // Backend Configuration backendUrl: 'https://your-backend.com', // Authentication auth: { token: 'your-jwt-token-here', tokenKey: 'my_chatbot_token', loginEndpoint: '/api/auth/login' }, // RAG Configuration enableRAG: true, ragThreshold: 0.7, maxRAGResults: 3, // UI Configuration theme: { primaryColor: '#007bff' }, welcomeMessage: 'Hello! I\'m your AI assistant with access to your documents.', // Content Ingestion (Optional) enableContentIngestion: false, ingestEndpoint: 'https://your-backend.com/api/ingest' });

🔧 Backend Requirements

✅ Your backend should have:
  • POST /search endpoint that accepts authentication
  • POST /api/auth/login endpoint for authentication
  • JWT token validation middleware
  • User-scoped document search (only return user's own documents)

📝 Example Backend Endpoints

Search Endpoint (Protected)

POST /search Headers: { "Authorization": "Bearer your-jwt-token", "Content-Type": "application/json" } Body: { "query": "user question", "max_results": 3, "threshold": 0.7 }

Login Endpoint

POST /api/auth/login Headers: { "Content-Type": "application/json" } Body: { "username": "user@example.com", "password": "password123" } Response: { "token": "jwt-token-here", "user": { "id": "user-id", "username": "user@example.com" } }

Minimal Setup

// 1. Include the chatbot script <script src="chatbot.js"></script> // 2. Initialize with your backend initRAGChatbot({ backendUrl: 'https://your-backend.com', auth: { token: 'your-jwt-token' } });
